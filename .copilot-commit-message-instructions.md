# Commit Message Instructions

## Format Guidelines
- Use imperative mood (e.g., "Add feature" not "Added feature")
- Keep the first line under 50 characters
- Capitalize the first letter
- No period at the end of the subject line
- Use the body to explain what and why vs. how

## Conventional Commit Types
- `feat:` New features
- `fix:` Bug fixes
- `docs:` Documentation changes
- `style:` Code style changes (formatting, etc.)
- `refactor:` Code refactoring
- `test:` Adding or updating tests
- `chore:` Maintenance tasks, dependencies

## Examples
```
feat: add MCP server configuration for enhanced AI capabilities

fix: resolve path mismatch in VS Code settings for Codespaces

docs: update README with Codespace setup instructions

chore: update devcontainer configuration for better Node.js support
```
